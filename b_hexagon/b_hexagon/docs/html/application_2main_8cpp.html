Detailed Description

Application entry point wiring ZeroMQ adapters to the CalculatorService.

Responsibilities performed here:

  1. Define runtime configuration (endpoints + PUB/SUB topics).
  2. Instantiate inbound (SUB) and outbound (PUB) adapters.
  3. Inject adapters into the application service (following Dependency Inversion).
  4. Start the blocking processing loop.

Hexagonal Architecture Note: Only composition occurs at the edge (in main). The domain/application layers remain agnostic to transport technology (ZeroMQ) â€“ we could substitute alternative adapters without altering business logic.
