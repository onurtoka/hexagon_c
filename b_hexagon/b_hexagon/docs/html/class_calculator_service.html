Detailed Description

Orchestrates the continuous receive -> compute -> send workflow.

Responsibilities:

  • Pull raw TrackData via IDataReceiver (inbound port).
  • Capture timing to compute delays using pure functions (TrackCalculations).
  • Emit enriched DelayCalculatedTrackData via IDataSender (outbound port).

Design Notes:

  • Adheres to the Dependency Inversion Principle; depends only on abstractions.
  • Currently runs an infinite loop; production hardening may introduce: graceful shutdown flag, exception handling & backoff, metrics, structured logging, batching.
  • Uses smart pointers for MISRA C++ compliance (Rule 15-1-1).

MISRA C++ 2023 Compliance:

  • Rule 15-1-1: Uses RAII with smart pointers instead of raw pointers
  • Rule 15-3-4: Exception safety specifications
  • Rule 12-1-4: Explicit constructor where appropriate
  • Rule 7-1-2: Const correctness applied throughout
