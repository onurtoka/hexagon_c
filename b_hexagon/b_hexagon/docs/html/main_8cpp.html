Application entry point demonstrating Hexagonal Architecture assembly.

This file demonstrates the Composition Root pattern where all dependencies are wired together at the application's entry point. It showcases how the hexagonal architecture components are assembled without coupling them together.

Responsibilities:

  1. Configuration: Define runtime settings (endpoints, topics, timeouts)
  2. Instantiation: Create all adapters and domain services
  3. Dependency Injection: Wire components following Dependency Inversion Principle
  4. Lifecycle Management: Start and manage the application lifecycle
  5. Error Handling: Handle startup and runtime errors gracefully

Hexagonal Architecture Benefits Demonstrated:

  • Testability: Each component can be easily mocked/stubbed
  • Flexibility: Adapters can be swapped without changing core logic
  • Maintainability: Clear separation of concerns
  • SOLID Principles: Dependency inversion applied throughout
