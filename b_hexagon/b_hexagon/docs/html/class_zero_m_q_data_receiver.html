Detailed Description

ZeroMQ SUB adapter for receiving TrackData published by a PUB socket (JSON format).

Expected payloads (topic optional): "{json}" OR "<topic> {json}"

JSON object must contain numeric fields (double / integer) for: trackId, xVelocityECEF, yVelocityECEF, zVelocityECEF, xPositionECEF, yPositionECEF, zPositionECEF, updateTime (ns), originalUpdateTime (ms), firstHopSentTime (ns)

Example (with topic 'track'): track {"trackId":12,"xVelocityECEF":10.1,"yVelocityECEF":0.0,"zVelocityECEF":-3.5, "xPositionECEF":1000.4,"yPositionECEF":2000.5,"zPositionECEF":3000.6, "updateTime":1712678899000000000,"originalUpdateTime":1712678898000000000, "firstHopSentTime":1712678897000000000}

NOTE: A minimal hand-written JSON parser is used to avoid bringing in a full dependency. It assumes: no nested objects, keys are unique, values are plain numbers, no whitespace requirements (whitespace is skipped). Missing or malformed fields raise std::runtime_error.
