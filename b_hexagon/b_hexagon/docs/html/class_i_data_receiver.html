Detailed Description

Abstraction for receiving a single TrackData instance from an external source.

Lifecycle / Usage:

  • Called repeatedly by the application service loop (CalculatorService).
  • Should block until data is available (or implement its own timeout / cancellation policy).
  • Any parsing / translation of external representation (JSON, CSV, binary) occurs inside the adapter.

Error Handling Guidelines:

  • Throw a domain-relevant exception (e.g., std::runtime_error) on unrecoverable parse / IO errors.
  • Prefer retry logic inside the adapter rather than leaking transient failures to the domain layer.

MISRA C++ 2023 Compliance:

  • Rule 15-3-4: Virtual destructor is noexcept by default
  • Rule 12-1-1: Interface should not be instantiated directly
