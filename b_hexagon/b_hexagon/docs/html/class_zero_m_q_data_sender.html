Detailed Description

ZeroMQ publisher adapter (PUB side) sending DelayCalculatedTrackData as JSON.

Rationale for switching from CSV to JSON:

  • Self-describing field names (better evolvability / backward compatibility)
  • Easier debugging (human-readable without schema knowledge)
  • Safer when adding optional fields (order no longer critical)

Example JSON (single line): {"trackId":42,"xVelocityECEF":1.1,"yVelocityECEF":2.2,"zVelocityECEF":3.3, "xPositionECEF":4.4,"yPositionECEF":5.5,"zPositionECEF":6.6, "updateTime":111, // ns "originalUpdateTime":222, // ms "firstHopSentTime":333, // ns "firstHopDelayTime":444, // ns "secondHopSentTime":555} // ns

If a topic is provided it is prepended: TOPIC {json}
